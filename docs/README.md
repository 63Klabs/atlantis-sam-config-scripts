# Getting Started

If you are part of an organization or class where you were told to use the SAM Config repository by a supervisor, instructor, or admin, then you are a *Developer*.

If you do not yet have a SAM Config repository installed to your personal or organization's account and you are in charge of installing for your own personal or organizational use, then you are an *Account Administrator*.

Do not clone the repository from the 63Klabs account. Administrators will create a copy and install in to an organizational account. Once installed by an admin, developers will clone it from the organization's account and pull and push infrastructure changes.

- [Account Administrators Start Here](./admin/00-Set-Up-AWS-Account-and-Config-Repo.md)
- [Developers Start Here](./00-Set-Up-Local-Environment.md)

## Next Steps for Developers

Now that you have your local environment set up, you can begin exploring the various application starters included in the SAM Config repository.

There are Atlantis application starters for deploying many serverless solutions.

To assist in getting developers up and running with the CLI scripts and application starters, various tutorials have been assembled and can be viewed at [Serverless Deployments using 63K Atlantis Tutorials](https://github.com/63Klabs/atlantis-tutorials).

Once you have explored the basic application starters, proceed to other application starters which include additional features such as advanced monitoring, caching, and resource configurations.
